# Image Processor - Specification

## Project Overview
- **Project name**: image-processor
- **Type**: Rust workspace with multiple crates
- **Core functionality**: C2PA (Coalition for Content Provenance and Authenticity) image processing with zkVM proofs
- **Target users**: Applications requiring image provenance verification and authenticity proofs

## Architecture

### Crates
- **lib**: Core C2PA library for extracting/manipulating C2PA manifests from images
- **app**: Host application that loads zkVM guest ELF and orchestrates processing
- **prover**: CLI that generates zkVM proofs for image authenticity
- **verifier**: CLI that verifies proofs generated by the prover
- **app-lib**: Shared library for app and prover (contains guest code)
- **zkVM Guest**: Embedded ELF that runs inside the prover's zkVM

### Data Flow
1. **Prover (Host)**: Extracts C2PA manifest, applies image operations, computes file hash
2. **zkVM (Guest)**: Receives image data and operations, applies operations internally, computes semantic hash
3. **Verification**: Both prover and zkVM compute hashes that must match

## Functionality Specification

### Core Features

#### 1. C2PA Manifest Extraction (lib)
- Extract C2PA manifests from JPEG and PNG images
- Parse manifest store and claims from image metadata
- Support for APP11 marker (JPEG) and specialized chunks (PNG)
- Extract certificate chains from manifests

#### 2. Image Operations
Image operations are applied in BOTH the prover (host) and zkVM (guest) to ensure correctness.

**Supported Operations:**
| ID | Operation | Parameters |
|----|-----------|------------|
| 0 | Crop | x, y, width, height |
| 1 | Resize | width, height |
| 2 | ResizeAspect | max_width, max_height |
| 3 | Brightness | value (i32) |
| 4 | Contrast | value (i32) |
| 5 | Exposure | value (f32) |
| 6 | Gamma | value (f32) |
| 7 | Thumbnail | max_size |
| 8 | CropResize | x, y, width, height, target_width, target_height |

#### 3. Image Editing (host-side)
- Use image-editor crate for actual image manipulation on host
- Applies operations to generate the output image file

#### 4. zkVM Guest Processing
- Receives original image hash and operation parameters as private inputs
- **Applies image operations internally** to compute output
- Computes semantic hash of the processed image
- Outputs: final_image_hash, num_operations

#### 5. Prover Flow
```
1. Extract C2PA from original image
2. Apply operations (host-side) to generate output image
3. Compute file hash of output image
4. Send data to zkVM:
   - has_manifest flag
   - C2PA metadata (issuer, timestamp, signature, claim_hash)
   - certificate_chain_verified flag
   - operation list (type + params)
   - final_image_hash (from host-side computation)
5. zkVM applies operations and computes its own hash
6. zkVM outputs: final_image_hash, num_operations
```

#### 6. Verifier Flow
- Receives proof and public values
- Validates the proof against known constraints
- Outputs: is_valid, final_image_hash, num_operations

## Dependencies
- **c2pa**: Official C2PA crate with file_io and unstable_api features
- **x509-parser**: X.509 certificate parsing
- **serde/serde_json/serde_cbor**: Serialization
- **sha2**: SHA-256 hashing
- **alloy-sol-types**: Solidity type definitions
- **pico-sdk**: For zkVM prover/verifier builds
- **image-editor**: Host-side image manipulation

## Technical Implementation
- Use official `c2pa` crate for manifest extraction
- CBOR parsing for manifest data
- Certificate chain verification following C2PA/X.509 semantics
- Entropy-based cryptographic data detection for smart extraction
- Host applies operations using image-editor crate
- zkVM guest computes semantic hash from operations

## Acceptance Criteria
1. All crates compile without errors
2. Can extract C2PA manifest from JPEG images
3. Can extract C2PA manifest from PNG images
4. Can parse certificate chains from manifests
5. Prover applies operations correctly on host
6. zkVM applies operations internally (semantic hash computation)
7. Prover generates valid zkVM proofs
8. Verifier correctly validates proofs
9. Handles images without C2PA manifests gracefully
10. File hash (prover) matches semantic hash (zkVM) for valid operations
